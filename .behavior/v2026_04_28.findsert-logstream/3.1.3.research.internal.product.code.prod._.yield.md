## pattern.2: generateLogMethod

**location:** src/domain.operations/generateLogMethod.ts:25-53 [2]

**what it does:**
- creates a single log method for a specific level
- checks if log level meets minimum threshold
- formats message via `formatLogContentsForEnvironment`
- outputs to `console.log` or `console.warn`

**relevance:** [EXTEND] — add outlet dispatch after console output

**citation [2]:**
```typescript
export const generateLogMethod = ({
  level,
  minimalLogLevel,
}: {
  level: LogLevel;
  minimalLogLevel: LogLevel;
}) => {
  return (message: string, metadata?: object) => {
    if (aIsEqualOrMoreImportantThanB({ a: level, b: minimalLogLevel })) {
      const consoleMethod = aIsEqualOrMoreImportantThanB({
        a: level,
        b: LogLevel.WARN,
      })
        ? console.warn
        : console.log;
      consoleMethod(
        formatLogContentsForEnvironment({
          level,
          timestamp: new Date().toISOString(),
          message,
          metadata,
        }),
      );
    }
  };
};
```

## pattern.3: formatLogContentsForEnvironment

**location:** src/domain.operations/formatLogContentsForEnvironment.ts:8-55 [3]

**what it does:**
- formats log content based on detected environment
- LOCAL: returns object with stringified metadata
- AWS_LAMBDA: returns JSON stringified object
- WEB_BROWSER: returns object with raw metadata

**relevance:** [REUSE] — outlets can use the structured log event before environment-specific format

**citation [3]:**
```typescript
export const formatLogContentsForEnvironment = ({
  level,
  timestamp,
  message,
  metadata,
}: {
  level: LogLevel;
  timestamp: string;
  message: string;
  metadata?: Record<string, any>;
}) => {
  const env = identifyEnvironment();
  if (env === SupportedEnvironment.LOCAL) {
    return { level, timestamp, message, metadata: JSON.stringify(metadata) };
  }
  if (env === SupportedEnvironment.AWS_LAMBDA) {
    return JSON.stringify({ level, timestamp, message, metadata });
  }
  if (env === SupportedEnvironment.WEB_BROWSER) {
    return { level, timestamp, message, metadata };
  }
  throw new Error('unsupported environment detected');
};
```

## pattern.8: identifyEnvironment

**location:** src/domain.operations/identifyEnvironment.ts:7-19 [8]

**what it does:**
- detects runtime environment: AWS_LAMBDA, WEB_BROWSER, LOCAL
- checks `AWS_LAMBDA_FUNCTION_NAME` env var for lambda
- checks `window` for browser

**relevance:** [REUSE] — outlets can use this to skip auto-collect environments

**citation [8]:**
```typescript
export const identifyEnvironment = (): SupportedEnvironment => {
  const isAwsLambdaEnvironment = !!process.env.AWS_LAMBDA_FUNCTION_NAME;
  if (isAwsLambdaEnvironment) return SupportedEnvironment.AWS_LAMBDA;
  const isWebBrowserEnvironment =
    typeof window !== 'undefined' && typeof window.document !== 'undefined';
  if (isWebBrowserEnvironment) return SupportedEnvironment.WEB_BROWSER;
  return SupportedEnvironment.LOCAL;
};
```
